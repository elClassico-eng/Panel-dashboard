// Основные понятия
1.blob - это объект, типа - файл
2.tree - это объект, типа - папки, которые создаются
3.commit - это объект, типа - commit, который позволяет нам сохранить новое состояние проекта
4.head - это ссылка на определенную "ветку -> commit" 
5.ветка - это ссылка на commit (эта ссылка автоматически перемещается на последний commit(чтобы у нас была всегда актуальная версия проекта))

// Alias
status: st, (показывает статус index-а проекта, те файлы, которые были изменены в текущей версии проекта)
checkout: ck, (переход по commit или по веткам проекта)
commit: com, (создание commit, который ссылается на tree)
branch: br, (выбрать ветку в которую хотел бы переместиться)
log: lg (история коммитов)

// Команды
git cat-file -t 0a34b - просмотр типа файла по hash
git cat-file -p 0a34b - просмотр содержимого файла по hash

// Правила создания:

feat: – добавление новой features
fix: – исправление ошибки
refactor: – рефакторинг кода (улучшение структуры без изменения логики)
chore: – технические изменения (обновление зависимостей, настройка CI/CD)
docs: – изменение документации
style: – исправление форматирования (пробелы, отступы, кавычки)
test: – добавление или обновление тестов
perf: – оптимизация производительности
